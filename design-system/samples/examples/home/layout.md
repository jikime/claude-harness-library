# home layout — source: https://moveuniversity.kr/ko

> 캡처: Playwright full-page, desktop 1440 / tablet 768 / mobile 390 (reference-*.png 동봉).
> 측정 신뢰도: 계측(스크린샷 + computed). 일부 섹션 수직 px는 추정.
> 성격: 단일 길이 랜딩(LP) — 견적 신청 전환 중심. 코랄(#ff876f) CTA가 페이지 전체에 반복 등장.

## 섹션 순서 (위→아래)

1. **top-nav** — `header` fixed, z-100, height ~107px, 배경 투명(히어로 위에 오버레이). 좌: 로고. 중: 텍스트 내비(이사·청소·자가진단·청소대학·꿀팁·인터뷰 등). 우: 전화번호 "1551-2425"(24px·700) + 코랄 CTA 버튼("무료 견적 신청"). max-width 1280 컨테이너, px 40.
2. **hero** — 풀폭 사진 배경(이사 현장) 위 중앙에 어두운 **라디얼 scrim**(rgba(0,0,0,0.75))를 깔고 흰 헤드라인. 배지("2024/2026 소비자 평가 대상 수상", 14px·600) → display 40px·700 2줄("1분만에 빠르게 / 간편 이사 견적 신청") → 코랄 CTA("1분 무료 견적 신청→", radius 12px). 우측에 떠있는 퀵 CTA 패널 시작(전화·일러스트).
3. **service-selector** — 흰 캔버스. H2 32px("어떤 이사를 준비하고 계신가요?") + 카테고리 아이콘/칩 행(이사·청소…, pill 버튼 active=#35363a / default=흰+보더). 하단에 빠른 견적 입력 바(원룸 등 옵션칩 + 전화 + 코랄 신청 버튼).
4. **dark-pitch** — 다크 패널(#35363a). 흰 텍스트 카피("무허가 업체와 고무줄 견적이 난무하는…") — 신뢰/문제제기 섹션.
5. **value-props (데이터/표준견적)** — 흰 배경, H3 24px·600 좌측 카피 + 우측 UI 일러스트(견적 화면 목업). "데이터 기반 표준 견적으로 부당한 추가금 차단" 류 3-pillar.
6. **service-grid** — 서비스 유형 카드 행(원룸/투룸이사 등, title-lg 28px·600). 카드 면색 + radius.
7. **checklist-promo** — 옅은 코랄 면(#fff5f3). H2 32px·700("이사 전, 이것만은 꼭 확인하세요! 필수 체크리스트") + "체크리스트 바로 가기" CTA(18px·700, radius 12px). 우측 일러스트.
8. **stats / counter** — 큰 숫자 카운터("12,323" 32px·600 등) 신뢰 지표 행.
9. **reviews** — 고객 리뷰 카드 그리드(별점 + 본문 14px·400/500 + 이름). 다수 카드 반복, flat(보더 기반).
10. **estimate-calculator** — "예상 견적 계산기"(H3 20px·700) 인터랙티브 입력 블록 + 결과.
11. **final-cta (다크 히어로)** — 다크 사진/패널 위 흰 헤드라인 40px·600("좋은 기사님을 만나는 이사, 이제 운이 아닙니다. 지금…") + 코랄 CTA(radius 16px).
12. **footer** — 다크(#35363a/#2b2b2b) 풀폭. 회사정보·약관·링크 컬럼, 회색 텍스트.
13. **floating quick-CTA** (오버레이, 섹션 아님) — 우측 fixed(z 98–99): 실시간 카톡상담(#fee500)·전화 1551-2425·무료견적 신청을 묶은 떠있는 패널. 항상 노출.

## 그리드 / 컨테이너
- 주 컨테이너 `max-width: 1280px`, 좌우 padding 40px(desktop). 보조 컨테이너 1200px / px 24px.
- 모바일 거터 24px로 축소.
- 콘텐츠 정렬: 헤드라인/카피 좌측 정렬 중심, 히어로·최종 CTA는 중앙 정렬.
- 카드 행: desktop 다열(서비스 3~4열, 리뷰 다열) → tablet 2열 → mobile 1열(스택).

## 여백 리듬
- 섹션 간 큰 수직 여백으로 구획(정확 px 추정 — 보강 권장). 흰 캔버스와 면색(다크 #35363a / 코랄틴트 #fff5f3 / 회색 #f4f5f7) 섹션을 번갈아 배치해 리듬 형성.
- 본문 단락 행간 1.6~1.7로 넉넉, 헤드라인 1.3~1.4.
- CTA는 페이지 전체에 4회+ 반복 등장(hero·selector·checklist·final + floating) — 전환 강조 패턴.

## 반응형 거동 (1440 / 768 / 390 관측)
- **nav**: desktop 가로 텍스트 내비 풀노출. tablet/mobile → 내비 항목이 압축되고 상단바에 전화/CTA만 강조, 아이콘 행으로 축약(모바일에서 카테고리 아이콘 가로 스크롤/그리드).
- **hero**: 사진+scrim 유지, 텍스트·CTA 폭 축소. mobile에서 헤드라인·CTA 세로 풀폭.
- **카드/그리드**: 다열 → 2열(tablet) → 1열 스택(mobile). 리뷰·서비스 카드 동일 패턴.
- **floating quick-CTA**: desktop/tablet 우측 떠있는 패널 → mobile 하단 고정바/축소 버튼군으로 전환(추정).
- **컨테이너**: 거터 40px(desktop) → 24px(mobile)로 축소, 섹션 면색은 풀블리드 유지.
- 색·타입 토큰은 브레이크포인트 간 유지(스케일 동일), 레이아웃만 재배치.

## 비고
- 면색 섹션(다크·코랄틴트) ↔ 흰 캔버스 교차가 핵심 합성 리듬.
- 카드 elevation은 거의 flat(그림자 대신 면색·hairline 보더로 분리).
- 정확 섹션 높이/수직 간격, hover 모션, 모바일 floating bar 정확 형태는 Known Gaps(tokens.md 참조).
