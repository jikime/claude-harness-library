# 구현 준비도 검증 매뉴얼 (readiness-check)

목적: 각 단계 산출물을 받아 **"다음 단계(또는 구현 에이전트)가 추가 질문 없이 진행할 수 있는가"**라는 단일 기준으로 회의적으로 게이트한다. 결과는 `readiness.md`에 누적한다. 생성자가 아니라 별도 평가자가 본다.

자기평가 편향을 전제로 한다 — 생성자는 자기 결과를 과대평가하므로, 통과는 근거가 있을 때만 준다. 판정은 가능한 한 객관 항목(존재·추적·문법 준수)에 묶는다.

## 게이트 1 — PRD readiness
- [ ] 모든 FR이 EARS 문법으로, 트리거·응답이 명시됐는가
- [ ] 모든 FR에 **테스트 가능한** Acceptance Criteria가 있는가 (모호하면 탈락)
- [ ] 내린 모든 가정이 §4 Assumptions에 기록됐는가 (몰래 가정 탐지 → 탈락)
- [ ] §19 Open Questions 블록이 존재하는가 (빈칸이면 "없음" 명시)
- [ ] §0 링크가 유효 경로를 가리키는가
- [ ] 서비스 유형(SaaS/AI/Agent)에 맞는 Special Rules가 적용됐는가
- [ ] §10/§11이 요약 수준인가 (상세를 PRD에 욱여넣지 않았는가)
- [ ] §16 Out of Scope로 범위 경계가 있는가

## 게이트 2 — PRD ↔ ARCHITECTURE 정합
- [ ] **추적성**: 모든 PRD FR-###이 ARCH 컴포넌트로 매핑되는가 (orphan FR → 탈락)
- [ ] 역방향: ARCH에 PRD 근거 없는 기능이 임의 추가되지 않았는가
- [ ] PRD §10/§11 요약이 상세 스키마·API 계약으로 전개됐는가
- [ ] 되돌리기 어려운 결정마다 ADR(대안·트레이드오프·confidence)이 있는가
- [ ] PRD 각 가정·미해결질문이 해소 또는 명시적 이월됐는가
- [ ] NFR이 "어떻게 달성"으로 구체화됐는가

## 게이트 3 — ARCHITECTURE ↔ TASKS 정합
- [ ] **추적성**: 모든 FR-### / 컴포넌트가 최소 한 태스크로 매핑되는가
- [ ] 모든 태스크가 Maps to(FR/컴포넌트)와 실행 가능한 Verification을 갖는가
- [ ] Agent Operating Guide 6영역(Commands/Testing/Structure/Style/Git/Boundaries)이 채워졌는가
- [ ] Boundaries의 ⚠️/🚫가 구체적인가 (최소 "시크릿 커밋 금지" 포함)
- [ ] 의존 순서·Phase가 모순 없는가

## 추적성 매트릭스 (모든 게이트 공통)
가능하면 표로 남긴다 — drift 방지의 핵심:
```
FR-### → ARCH 컴포넌트 → T-### → (구현 리뷰 시) 코드
```
양방향 orphan(매핑 없는 FR, 근거 없는 컴포넌트/태스크)을 탈락 사유로 본다.

## 실패모드 가드 (반드시 탐지)
- **막연한 AC**: "직관적", "빠르게", "잘 동작" → 측정 불가 → 탈락
- **몰래 가정**: 스택·범위·유형을 가정해놓고 §4에 미기록 → 탈락
- **boundaries 누락**: TASKS에 ⚠️/🚫 없음 → 탈락
- **과대 컨텍스트**: 한 태스크에 너무 많은 책임을 몰아넣음 → 분할 권고
- **사람 리뷰 생략**: 위험 행동(merge/배포/결제/삭제)을 사람 게이트 없이 통과시킴 → 탈락

## 판정 예시 (기준 고정)
- **통과**: "FR-003: When 사용자가 PDF를 업로드하면, the system shall 10초 내 요약을 반환한다. AC: 5MB PDF 입력 시 200 응답 + summary 필드, 11초 초과 시 타임아웃 에러." → 트리거·응답·측정 가능 AC 있음.
- **탈락**: "FR-003: 사용자가 파일을 올리면 잘 처리한다." → 트리거 모호, 응답·AC 측정 불가.

## 출력 — `readiness.md`에 누적
```
## [게이트 N] {산출물} — 판정: 통과 / 탈락 (라운드 k)
- 통과 항목: …
- 탈락 항목 + 구체 위치 + 수정 지시: …
- 추적성 매트릭스: …
```
탈락 시 구체적 위치와 수정 지시를 적어 생성자가 바로 고칠 수 있게 한다.
