# 보안 체크리스트 적용 매뉴얼

## 트리거
code-security-reviewer 또는 supply-chain-deploy-verifier가 현재 프로젝트를 보안 검토할 때 따른다. 단일 진실 원천(SSOT)은 이 스킬에 동봉된 `references/dev-security-checklist.md`(스킬과 함께 배포되는 정본)이며, 이 SKILL.md는 그 체크리스트를 **어떻게 판정·증거화·저장**할지를 규정한다(기준 항목은 references 정본을 읽는다).

## 판정 4상태 (모든 항목은 반드시 하나로)
| 상태 | 의미 | 필수 증거 |
|---|---|---|
| **통과** | 기준을 만족 | 확인한 파일/설정 또는 도구 통과 결과 |
| **위반** | 기준 미달, 위험 존재 | `파일:라인` 또는 도구 출력 + 무엇이 왜 위험한지 |
| **N-A** | 이 프로젝트에 해당 차원이 없음 | 왜 적용되지 않는지 한 줄(예: "컨테이너 없음", "DB 미사용") |
| **미검증** | 도구 부재·접근 불가로 확인 못 함 | 무엇을 못 했고 무엇이 있어야 하는지 |

**원칙: 억지로 채우지 않는다.** 적용되지 않으면 N-A, 확인 못 하면 미검증. 추측으로 통과/위반 처리 금지(거짓 리포트 방지).

## 하이브리드 도구 실행 (비용 게이트)

무거운 스캐너(`semgrep`, `trivy fs`)를 레포 전체에 돌리면 느리고 토큰을 많이 쓴다. **기본은 변경분 한정이고, 레포 전수 스캔은 종합 모드이거나 사용자 승인이 있을 때만 한다.**

1. **스코프 우선** — 변경분 모드면 도구를 **변경 파일에만** 건다(`semgrep <changed files>`, `gitleaks` 스테이징/diff 범위). 전수 스캔(`semgrep .`, `trivy fs .`)은 종합 모드 또는 명시 승인 시에만.
2. **있으면 실행(스코프 안에서)** — 결과를 증거로 인용한다.
   - 코드/시크릿: `gitleaks detect --no-banner`, `semgrep --config p/security-audit --config p/owasp-top-ten --severity ERROR`
   - 공급망/배포: `trivy config .`, (의존성) `trivy fs --scanners vuln`, (이미지가 있을 때만) `trivy image`, `cosign verify ...`. 이미지·서명 산출물이 없으면 돌리지 말고 **N-A**.
3. **없거나 실패하면 읽어서 판정** — 코드·`Dockerfile`·`.github/workflows`·매니페스트·lockfile을 직접 읽어 패턴으로 판정하고, 도구 미실행 항목은 **미검증**(도구 부재) 또는 **위반/통과**(파일 근거 확실 시)로 표기한다.
4. 도구 설치를 시도하지 않는다(환경 변경 금지). 미설치는 미검증으로 남긴다.

## 검사 대상 범위 (모드)
- **변경분 모드(기본)** — `git diff --name-only` 대상 파일만. 빠른 PR 점검.
- **전수 모드** — 레포 전체 + CI 설정. 릴리스 전.
오케스트레이터가 모드를 알려준다. 모르면 변경분으로 가정하고 명시한다.

## 차원 매핑 (어느 에이전트가 어느 절을 보나)
- **code-security-reviewer** → 체크리스트 "코드 작성 보안 기준"(입력검증·인증/인가·SQL·명령/파일·SSRF·XSS·시크릿·로그·암호화·API) + "코드 리뷰에서 바로 차단할 패턴" + "PR 보안 체크리스트"
- **supply-chain-deploy-verifier** → "빌드·배포·공급망 판정 기준"(SCA·IaC·이미지·서명/출처·CI 공급망·SBOM) + "릴리스 산출물 확인"

## 산출물 형식 (findings 파일)
각 검토자는 자기 영역만 담는다. 형식:

```markdown
# {코드|공급망} 보안 발견  (모드: 변경분|전수 / 일시는 오케스트레이터가 stamp)

## 요약
- 통과 N · 위반 N · N-A N · 미검증 N

## 차원별 판정
### {차원명}
- 상태: 위반
- 증거: path/to/file.py:42  (또는 `semgrep` 규칙 ID/출력)
- 내용: 무엇이 왜 위험한가 (위험도 등급은 매기지 않음 — rater가 함)
- 권장: 한 줄 수정 방향
```

## 품질 체크
- [ ] 모든 항목이 4상태 중 하나로 판정됐다
- [ ] 위반·미검증에 증거가 있다
- [ ] N-A에 사유가 있다
- [ ] 위험도 등급·예외 승인·go/no-go를 여기서 결정하지 않았다(rater 몫)
- [ ] findings 파일을 지정 경로에 저장했다(요약으로 대체하지 않음)
