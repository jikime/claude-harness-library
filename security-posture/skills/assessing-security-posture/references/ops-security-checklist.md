# 운영 보안 태세 체크리스트 (Operational Security Posture Checklist)

> **목적** — 운영 중인 서비스의 보안 태세를 도메인별로 평가하는 단일 진실 원천(SSOT).
> **대상** — `security-posture` 하네스가 이 체크리스트로 충족/미흡/N-A/미검증을 판정한다.
> **범위** — 운영(런타임·인프라·거버넌스) 보안. 개발·배포 시점 보안은 별도 `pre-deploy-gate` 하네스(정본: `reviewing-predeploy-security/references/dev-security-checklist.md`)가 담당한다.
> **프레임워크 매핑** — NIST CSF 2.0 / CIS Controls v8 / ISO 27001 / SOC 2 / PCI-DSS.

판정 4상태: **충족**(통제 작동) · **미흡**(통제 부재/약함) · **N-A**(해당 서비스에 없음) · **미검증**(증거·접근 부재로 확인 못 함). 억지로 채우지 않는다.

---

## ① 클라우드·인프라·네트워크 (cloud-infra-network)

### 클라우드 설정·자세 (CSPM)
- [ ] CIS 벤치마크 기준 클라우드 계정 설정 점검이 정기적이다 (NIST PR.IP / CIS 4)
- [ ] 퍼블릭 노출 스토리지(S3/GCS/Blob)가 의도된 것만 공개다
- [ ] 루트/관리 계정에 MFA, 일상 사용 금지가 적용된다
- [ ] CloudTrail/Activity Log 등 감사 로그가 활성·보존된다
- [ ] 리소스 태깅·자산 인벤토리가 최신이다

### IAM·권한
- [ ] 최소권한 원칙, 와일드카드 권한·과도한 역할이 없다 (CIS 5 / NIST PR.AC)
- [ ] 미사용 자격증명·액세스 키가 회전·폐기된다
- [ ] 서비스 계정/머신 ID가 인벤토리·관리된다

### 네트워크
- [ ] 보안 그룹/방화벽이 최소 개방 (0.0.0.0/0 무분별 개방 없음) (CIS 4 / NIST PR.AC-5)
- [ ] 네트워크 세그먼테이션/마이크로세그먼테이션이 적용된다
- [ ] 전송 구간 TLS 1.2+ 강제, 약한 cipher 비활성
- [ ] 외부 노출 앱에 WAF, DDoS 완화가 있다
- [ ] 관리 접근은 VPN/bastion/zero-trust로 제한된다

## ② 위협 탐지·모니터링 (threat-detection-monitoring)

### 로그·SIEM
- [ ] 핵심 시스템 로그가 중앙 수집·상관분석된다 (NIST DE.AE / CIS 8)
- [ ] 탐지 규칙(인증 이상·권한 상승·데이터 유출)이 운영된다
- [ ] 로그 보존 기간이 컴플라이언스 요건을 충족한다

### 엔드포인트·런타임
- [ ] EDR/안티멀웨어가 엔드포인트·서버에 배포된다 (NIST DE.CM / CIS 10)
- [ ] 컨테이너/클라우드 런타임 위협 탐지(Falco 등)가 있다
- [ ] 파일 무결성·드리프트 모니터링이 있다

### 대응 연계
- [ ] 알림이 SOC/온콜로 전달되고 우선순위가 매겨진다
- [ ] UEBA/이상행위 탐지가 내부 위협을 커버한다
- [ ] 알림 피로(오탐) 관리·튜닝 절차가 있다

---

## ③ ID·접근·데이터 보호 (identity-data-protection)

### 인증·접근
- [ ] 전사 MFA, SSO가 적용된다 (NIST PR.AA / CIS 6)
- [ ] 특권 접근 관리(PAM)·세션 모니터링이 있다
- [ ] 정기 접근 검토(access recertification)가 수행된다
- [ ] 퇴사자·역할 변경 시 적시 권한 회수가 된다
- [ ] 비밀번호 정책·시크릿 관리(Vault/KMS)가 표준화됐다

### 데이터 보호
- [ ] 저장 데이터 암호화(at-rest)와 키 관리(KMS/HSM)가 있다 (NIST PR.DS / CIS 3)
- [ ] 전송 데이터 암호화(in-transit)가 강제된다
- [ ] DLP로 민감정보 유출이 모니터링된다
- [ ] 개인정보(PII/PHI) 분류·처리·보존 정책이 있다 (GDPR/개인정보보호법)
- [ ] 데이터 접근이 최소권한·감사된다

---

## ④ IR·복원력·컴플라이언스 (ir-resilience-compliance)

### 인시던트 대응
- [ ] IR 플레이북·에스컬레이션 매트릭스가 있다 (NIST RS / CIS 17)
- [ ] 정기 IR 훈련(테이블탑)이 수행된다
- [ ] 포렌식 수집·증거 보존 절차가 있다

### 복원력·연속성
- [ ] 백업이 자동화·암호화·오프사이트/불변(immutable)이다 (NIST RC / CIS 11)
- [ ] 백업 복구 테스트가 정기적으로 검증된다
- [ ] 랜섬웨어 대비(격리·복구·킬스위치)가 있다
- [ ] DR/BCP 계획과 RTO/RPO가 정의·검증된다

### 컴플라이언스·거버넌스
- [ ] 적용 프레임워크(ISO 27001/SOC 2/PCI-DSS/NIST CSF) 갭이 추적된다
- [ ] 보안 정책·표준이 문서화·승인·주기 검토된다
- [ ] 제3자/벤더 리스크 관리가 있다
- [ ] 보안 인식 교육이 정기적이다

> 이 절의 항목은 GRC 프로그램 사실이라 기술 증거(IaC·설정)로는 검증되지 않는다. 관련 문서(정책·교육 기록·벤더 평가·갭 트래커)가 **제공될 때만** 평가하고, 없으면 **미검증**으로 표기한다(추측으로 충족 처리 금지).

## 성숙도 모델 (스코어카드 기준)
도메인별로 0–5 또는 NIST CSF Tier로 평가한다.

| 수준 | 의미 |
|---|---|
| 0 None | 통제 없음 |
| 1 Initial | 임시적·문서화 안 됨 |
| 2 Developing | 일부 도입, 비일관 |
| 3 Defined | 표준화·문서화 |
| 4 Managed | 측정·모니터링 |
| 5 Optimizing | 지속 개선·자동화 |

**리메디에이션 우선순위** = (갭 위험 × 노출/영향) ÷ 구현 난이도. Critical 갭은 즉시, High는 분기 내.

## 적용 원칙
- 운영 인프라/클라우드가 없는 대상에선 다수 도메인이 **N-A**. 정직히 표기한다.
- 라이브 접근(클라우드 CLI 등)이 없으면 IaC·설정·문서 증거로 판정하고, 확인 불가는 **미검증**.
- 이 체크리스트는 평가 기준이며, 실제 회계 감사·법적 인증 발급을 대체하지 않는다.
