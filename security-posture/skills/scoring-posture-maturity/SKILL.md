# 태세 성숙도 채점 매뉴얼

## 트리거
posture-maturity-synthesizer가 4개 도메인 파일(`01`~`04`)을 통합 채점할 때 따른다. 성숙도 모델 정본은 같은 하네스로 함께 배포되는 `assessing-security-posture` 스킬의 `references/ops-security-checklist.md` 성숙도 모델.

## 1단계 · 도메인 성숙도 점수
도메인별로 0–5(또는 NIST CSF Tier 1–4)로 채점한다. **수준 정의(0 None ~ 5 Optimizing)는 정본인 `assessing-security-posture`의 `references/ops-security-checklist.md` 성숙도 모델을 따른다**(여기서 재정의하지 않는다).

채점 규칙(회의적):
- 충족 비율 + 증거 강도로 산정한다.
- **미검증은 점수를 올려주지 않는다.** 미검증이 많으면 신뢰구간을 낮추고 그 사실을 명시한다.
- N-A는 분모에서 제외한다(없는 통제로 감점하지 않음).
- 불확실하면 보수적으로 한 단계 낮게.

## 2단계 · 갭 우선순위
`우선순위 = (갭 위험 × 노출/영향) ÷ 구현 난이도`
- **Critical** — 즉시(인터넷 노출·미인증·데이터 유출 직결)
- **High** — 분기 내
- **Medium/Low** — 백로그

## 3단계 · 리메디에이션 백로그
갭별로 권고·우선순위·예상 난이도·연관 도메인을 정리한다. 도메인 간 겹치는 갭은 한 번만.

## 4단계 · 전체 태세 요약
종합 성숙도(도메인 평균 또는 최저 도메인 강조) + 가장 시급한 Top 3~5 갭 + 신뢰도(미검증 비율).

## 산출물 형식 (`posture-scorecard.md`)
```markdown
# 운영 보안 태세 스코어카드

## 전체 태세: 종합 성숙도 N.N / 5  (신뢰도: 미검증 N%)
가장 시급한 갭: 1) ... 2) ... 3) ...

## 도메인별 성숙도
| 도메인 | 점수 | 충족/미흡/N-A/미검증 | 핵심 갭 |
|---|---|---|---|
| 클라우드·인프라·네트워크 | 2/5 | ... | ... |
| 위협 탐지·모니터링 | ... | ... | ... |
| ID·접근·데이터 보호 | ... | ... | ... |
| IR·복원력·컴플라이언스 | ... | ... | ... |

## 리메디에이션 백로그 (우선순위순)
| 우선순위 | 갭 | 도메인 | 권고 | 난이도 |
|---|---|---|---|---|
| Critical | ... | ① | ... | 중 |

## 미검증으로 남은 영역 (사람 확인 필요)
- {항목} — 무엇이 있어야 확인되는가
```

## 품질 체크
- [ ] 모든 도메인에 성숙도 점수를 부여했다
- [ ] 미검증을 충족처럼 점수화하지 않았다(신뢰도 명시)
- [ ] N-A를 감점 분모에서 제외했다
- [ ] 갭을 우선순위화하고 백로그를 만들었다
- [ ] 새 평가를 만들지 않고 입력만 종합했다
- [ ] 컴플라이언스 "달성"을 단정하지 않았다(준비도까지)
- [ ] 스코어카드를 지정 경로에 저장했다
